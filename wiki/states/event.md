# Event — 이벤트/팝업

## 화면 특징
- 다양한 팝업/대화상자 오버레이
- "확인"/"취소" 또는 "닫기" 버튼
- 이벤트 텍스트/보상 아이콘

## 감지 방법
- 반투명 오버레이 + 중앙 팝업 창
- "확인", "취소", "닫기" 버튼 텍스트
- 보상 아이콘 (두카트, 아이템 등) 존재 여부

## 전이 조건

| 조건 | 대상 상태 |
|------|----------|
| "확인" 버튼 탭 | 이전 상태 (sailing/in-port 등) |
| "취소" 버튼 탭 | 이전 상태 |
| 보상 수령 완료 | 이전 상태 |
| 알 수 없는 팝업 | recovery |
| 점검/업데이트 알림 | stopped |

## 하위 상태
- **popup-generic**: 일반 알림 팝업 (시스템 메시지, 공지 등)
- **popup-reward**: 보상 수령 팝업 (전투 보상, 과제 완료, 이벤트 보상)
- **popup-trade-event**: 교역 이벤트 (폭등/폭락/유행/과잉/폭증 알림)
- **popup-disaster**: 재해 발생 팝업 (화재, 침수, 전염병, 폭풍 등)
- **popup-error**: 오류 팝업 (네트워크 오류, 서버 오류 등)
- **popup-maintenance**: 점검 안내 (서버 점검, 긴급 점검)
- **popup-update**: 업데이트 필요 안내

## 주의사항
- 팝업 종류에 따라 처리 방식 상이: 대부분 "확인" 탭으로 해소
- 재해 팝업: 도구 사용 필요 (소화모래, 통, 라임주스, 징벌의 밧줄 등)
- 교역 이벤트(유행/폭등)는 교역 기회 → 무시하지 말 것
- 점검/업데이트 팝업은 자동 복구 불가 → stopped 전이
- 오류 팝업 반복 시 recovery 진입
- 부관 위임 설정 시 오프라인 중 재해/강습 자동 대응
