# Stopped — 정지

## 화면 특징
- 특정 화면 없음 (봇 자체가 동작 중단)
- 마지막 화면이 그대로 유지됨

## 감지 방법
- recovery 상태에서 3회 복구 시도 실패
- 점검/업데이트 팝업 감지
- 수동 정지 명령 수신

## 전이 조건

| 조건 | 대상 상태 |
|------|----------|
| 사용자 resume 명령 | sailing 또는 in-port (상황에 따라) |
| 사용자 stop 확인 | stopped 유지 |

## 정지 사유
- recovery 3회 실패 (자동 복구 불가)
- 서버 점검 / 긴급 점검
- 앱 업데이트 필요
- 네트워크 장기 단절
- 사용자 수동 정지

## 알림 발송
- Telegram 알림: 정지 사유, 마지막 상태, 타임스탬프
- 마지막 스크린샷 첨부 (가능한 경우)

## 주의사항
- 자동 재시작 없음 — 반드시 사람 개입 필요
- 정지 상태에서는 어떤 자동화 액션도 수행하지 않음
- 사용자가 원인 확인 후 resume 명령으로 재개
- 점검 종료 / 업데이트 완료 후에만 재개 가능
