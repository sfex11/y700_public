"핸드폰 화면이 보여"
Termux에서 AI 에이전트 첫 동작

2026-04-13 · 젬대버 프로젝트 · 읽기 5분

4월 12일 토요일 밤. PokeClaw이라는 오픈소스를 발견했다. 안드로이드 앱으로 Kotlin 기반, Gemma 4를 LiteRT-LM으로 구동해서 게임을 자동화하는 프로젝트다. 재미있어 보였지만 하나 문제가 있었다 — Termux에서 빌드가 안 된다.

Android SDK가 필요하고, Gradle이 필요하고, 결국 Android Studio가 필요하다. 태블릿에서 그걸 할 수는 없다.

그래서 결심했다. 직접 만든다.

"아키텍처는 간단하게"

요구사항은 하나다. API 키 없이 오프라인에서 완전 자율 동작. 서버에 의존하지 않고, 인터넷이 끊겨도 돌아가는 에이전트.

구성은 이렇게 정리됐다:

구성요소선택비고
제어 언어Python 3.13유연한 프로토타이핑
화면 제어ADBscreencap, input tap/swipe
LLMGemma-4-E4B-it Q4_K_M멀티모달, function-calling
LLM 엔진llama.cpp (서버 모드)OpenAI 호환 API
통신Telegram Bot원격 명령/결과 수신

하드웨어는 내 Y700 태블릿. Snapdragon 8850P에 Adreno 840 GPU, 14GB RAM. 충분하다.

"Termux 루트가 꽉 찼어"

시작하자마자 첫 번째 장애물. Termux 루트 파일시스템이 100%였다. 0바이트.

사용 가능한 공간: 0바이트.
이건 좀...

캐시를 비우고, 불필요한 패키지를 지우고, 프로젝트 디렉토리를 정리했다. 간신히 숨통이 트였다.

pkg install android-tools build-essential cmake git
pkg install python python-telegram-bot pyyaml requests httpx
pip install numpy Pillow

opencv-python은 pip에서도 termux에서도 빌드 실패. 결국 PIL + numpy 조합으로 가야 했다. 나중에 알게 되지만, 이게 오히려 다행이었다.

"adb tcpip 5555"

내 태블릿에서 내 태블릿을 제어하는 기분은 묘했다. ADB 무선 디버깅으로 자기 자신에게 연결한다.

adb tcpip 5555
adb connect 127.0.0.1:5555
adb shell screencap -p /sdcard/screen.png
adb pull /sdcard/screen.png ./screenshots/

화면 캡처 → 이미지 저장 → Python에서 읽기. 기본 루프의 뼈대가 됐다.

"llama.cpp 빌드가 안 돼"

이게 이 프로젝트의 본격적인 고비였다. llama.cpp를 Android에서 빌드하려니 spawn.h가 없다. Android Bionic libc에 POSIX spawn이 구현 안 돼 있다.

⚠️ 빌드 에러: spawn.h not found

sheredom/spawn.h 스텁을 넣었는데, 최소 스텁으로는 안 된다. posix_spawn_file_actions_addclose 같은 함수들까지 전부 스텁해야 한다.

결국 전체 스텁 함수를 포함한 헤더를 직접 작성했다. 그리고야 llama-server 빌드에 성공.

"5.4GB 모델 다운로드 완료"

Hugging Face에서 Gemma-4-E4B-it의 Q4_K_M 양자화 모델을 다운받았다.

파일크기용도
google_gemma-4-E4B-it-Q4_K_M.gguf5.4GB언어 모델
mmproj-F16.gguf990MB비전 인코더 (멀티모달)

합계 6.4GB. 450GB 남은 저장공간에서는 문제없었다.

"첫 POC가 돌아갔다"

마침내 기본 루프를 완성했다:

  1. 캡처 — ADB로 화면 스크린샷
  2. 분석 — LLM(Vision)에 이미지를 넘겨서 현재 상황 파악
  3. 전송 — 텔레그램으로 분석 결과 발송
  4. 수신 — 사용자가 텔레그램으로 명령 입력
  5. 실행 — ADB로 tap/swipe 실행
✅ POC v1 동작 확인

기본 루프 성공. 화면을 보고 → 텔레그램으로 알려주고 → 내 명령을 받아 → 화면을 터치한다. API 키 제로, 인터넷 없이도 동작.

다만 한 가지 문제가 있었다. CPU 전용이라 비전 분석 한 번에 약 106초가 걸렸다. 2분 가까이 기다려야 하는 건 실시간 게임 자동화에 쓸 수 없는 속도다.

GPU 가속이 필요했다. 그건 다음 편에.

배운 것

  • Android Bionic libc에는 POSIX spawn이 없다 → 스텁 헤더 필요
  • OpenCV 없이도 PIL + numpy로 화면 인식 기본 기능은 가능하다
  • ADB 무선 디버깅으로 같은 기기를 제어할 수 있다
  • llama.cpp 서버 모드는 OpenAI API 호환이라 기존 클라이언트 코드를 재사용할 수 있다
  • CPU 전용 LLM은 비전 분석에 실시간으로 쓰기엔 너무 느리다
